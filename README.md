# Machine_Learning_Course
Я проходила курси Intro to Machine Learning, Intermediate Machine Learning та Machine Learning Explainability на платформі Kaggle. Я обрала саме ці курси, тому що мене давно цікавить сфера machine learning та штучного інтелекту загалом. Зараз ML використовується практично всюди: у соціальних мережах, рекомендаційних системах YouTube, TikTok чи Netflix, у медицині для прогнозування хвороб, у банках для аналізу ризиків, у системах розпізнавання облич, безпілотних автомобілях та навіть у біотехнологіях. Мені дуже цікаво не просто користуватись такими технологіями, а розуміти, як вони працюють всередині, як модель “вчиться” на даних, чому вона приймає певні рішення та як можна зробити її точнішою. Особливо мене зацікавила тема explainability, тобто пояснення рішень моделей, тому що зараз ШІ починає впливати на дуже важливі сфери життя людей, і недостатньо просто отримати prediction — потрібно ще й розуміти, чому модель дала саме такий результат. Також мені подобається, що machine learning поєднує програмування, математику, логіку та аналіз даних. Для мене це одна з найцікавіших сфер ІТ, оскільки вона постійно розвивається і має величезну кількість реальних застосувань.

Під час проходження курсів я навчилась працювати з великою кількістю інструментів та методів machine learning. Спочатку я вивчила основи роботи з даними у pandas: як завантажувати датасети, обробляти їх, вибирати потрібні features та працювати з DataFrame. Далі я навчилась будувати перші ML-моделі за допомогою scikit-learn. Я працювала з Decision Trees, Random Forest та XGBoost і зрозуміла різницю між цими моделями, їхні переваги та недоліки. Також я навчилась правильно ділити дані на training та validation sets, використовувати train_test_split та оцінювати якість моделей через MAE і cross-validation. Окремо я вивчила проблему overfitting та underfitting і зрозуміла, чому модель може показувати майже ідеальні результати на training data, але погано працювати на нових даних. Дуже корисною для мене була тема preprocessing даних. Я навчилась працювати з missing values, використовувати SimpleImputer, one-hot encoding та ordinal encoding для categorical features, а також будувати pipelines і ColumnTransformer, щоб автоматизувати preprocessing та тренування моделі. Також я навчилась підбирати параметри моделей та порівнювати їхню ефективність.

Найцікавішим(але складним) для мене був курс Machine Learning Explainability. До цього я не сильно сприймала ML-моделі: модель видає prediction, але незрозуміло, чому саме. Під час цього курсу я навчилась використовувати permutation importance, partial dependence plots та SHAP values для пояснення роботи моделей. Особливо цікавими були SHAP values, тому що вони дозволяють побачити, які саме features вплинули на prediction для конкретного об’єкта. Наприклад, у задачі з прогнозування повторної госпіталізації пацієнтів можна було побачити, які фактори найбільше підвищують ризик readmission. Мені дуже сподобалось, що explainability дозволяє не просто створити модель, а ще й зрозуміти її логіку та перевірити, чи вона робить адекватні висновки. Також було цікаво працювати з partial dependence plots і аналізувати, як зміна окремого feature впливає на prediction моделі.

Найскладнішими для мене були останні завдання з Intermediate Machine Learning та Explainability. Там уже потрібно було не просто повторити код із прикладу, а самостійно розуміти логіку моделей, аналізувати результати та робити власні висновки. Особливо складними були SHAP summary plots та dependence plots, тому що спочатку було важко зрозуміти, як правильно інтерпретувати всі точки, кольори та вплив features на prediction. Також важкими були моменти, де потрібно було розуміти математичну частину моделей, а не лише писати код. Наприклад, деякі формули з експонентою e^x спочатку були для мене незрозумілими, і довелося додатково розбиратись, як саме модель використовує ці залежності.
Крім цього, складною була велика кількість графіків та їх аналіз. У курсі потрібно було будувати permutation importance plots, partial dependence plots, SHAP summary plots, dependence plots та інші візуалізації, а потім пояснювати, що саме вони показують і чому модель поводиться саме так. Інколи було складно зрозуміти, чому модель показує хороші результати на training data, але гірші на validation data, або чому feature має високу importance, але partial dependence plot показує дуже малу зміну prediction. Також непростими були завдання, де потрібно було самостійно покращувати модель та знижувати MAE через tuning параметрів. Проте саме ці складні теми дали мені найбільше нового досвіду та допомогли значно краще зрозуміти machine learning та роботу моделей на практиці.

Беззмістовного матеріалу для мене було небагато, але деякі частини Intro to Machine Learning здавалися занадто простими та трохи затягнутими. Наприклад, початкові завдання з basic pandas operations, створення найпростішої моделі чи дуже елементарні пояснення train/test split були для мене досить очевидними, тому що частину цього я вже знала раніше. Також деякі вправи повторювали одна одну: наприклад, потрібно було багато разів запускати схожий код для тренування моделей чи обчислення MAE, через що частина завдань уже не давала багато нових знань. Інколи трохи беззмістовними здавалися дуже дрібні зміни параметрів у моделях, коли потрібно було декілька разів запускати майже однаковий код лише для того, щоб отримати трохи інше число MAE. У таких моментах більше часу йшло на технічне повторення дій, ніж на реальне розуміння нової теми. Також деякі дуже прості графіки або приклади були надто очевидними після складніших тем із explainability та SHAP values. Проте навіть ці частини були корисними для закріплення бази, тому я б не сказала, що курс мав справді “непотрібний” матеріал. Швидше, деякі теми були занадто базовими порівняно з більш складними й цікавими частинами курсу. Також мені трохи не вистачило більшої кількості тем про Random Forest(в Intro to Machine Learning)  та інші складніші моделі, оскільки саме вони були для мене одними з найцікавіших під час проходження курсу.
